12 API

Djula external symbols documentation

Common Lisp Function: (compile-template*name)

Compiles template NAME with compiler in `CURRENT-COMPILER'

Common Lisp Function: (url-encodestring)

URL-encodes a string using the external format EXTERNAL-FORMAT.

Common Lisp Variable: *current-store*

The currently in-use template store. Defaults to a FILE-STORE.

Common Lisp Variable: *allow-include-roots*
Common Lisp Variable: *current-compiler*
Common Lisp Variable: *fancy-debug-p*

When enabled, displays fancy html based debugging information for the {% debug %} tag

Common Lisp Variable: *djula-execute-package*
Common Lisp Function: (fetch-template*key)

Return the text of a template fetched from the `CURRENT-STORE'.

Common Lisp Function: (url-encode-pathpath)
Common Lisp Function: (url-decode)
Common Lisp Macro: (def-tag-compilername args &body body)
Common Lisp Function: (find-template*name &optional (error-p t))
Common Lisp Variable: *fancy-error-template-p*

When enabled, show a fancy template when an error ocurrs

Common Lisp Variable: *default-language*
Common Lisp Function: (render-template*template &optional stream &rest *template-arguments*)

Render TEMPLATE into STREAM passing `TEMPLATE-ARGUMENTS'

Common Lisp Variable: *default-template-arguments*
Common Lisp Function: (add-template-directorydirectory &optional (template-store *current-store*))

Adds DIRECTORY to the search path of the TEMPLATE-STORE

Common Lisp Variable: *catch-template-errors-p*

When enabled, caught errors during the rendering of the template are written to the output instead of being handled by the lisp listener

Common Lisp Variable: *current-language*
Common Lisp Variable: *verbose-errors-p*

When enabled, errors are displayed more verbosely. Good for debugging
